# SnapMD 一键打包工具使用说明

## 📦 脚本文件说明

本项目提供的 Windows 一键打包脚本默认生成免安装便携版 `.exe`，双击即可运行，不生成 MSI/NSIS 安装器。

| 脚本文件 | 平台 | 说明 |
|---------|------|------|
| `build-win.ps1` | Windows | 生成便携版 exe（推荐，彩色输出） |
| `build-win.bat` | Windows | 生成便携版 exe（兼容性最好） |
| `build-mac.sh` | macOS | Shell 脚本版本 |

## 🚀 使用方法

### Windows 平台

如果使用终端命令，推荐先进入仓库根目录：

```powershell
cd D:\GitWork\MMY_SnapMD
npm run tauri:dev:image   # 启动桌面测试样本
npm run tauri:build:portable  # 生成免安装单文件 exe
npm run tauri:build           # 生成 MSI/NSIS 安装包
```

Windows 安装包已配置 WiX 语言为 `zh-CN`，用于避免中文产品名在 MSI 默认 `en-US` / codepage 1252 下打包失败。

#### 方法一：使用 PowerShell（推荐）

1. 右键点击 `build-win.ps1`
2. 选择"使用 PowerShell 运行"
3. 如果提示执行策略错误，以管理员身份运行 PowerShell，执行：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
4. 然后重新运行脚本

#### 方法二：使用批处理

1. 双击 `build-win.bat`
2. 等待脚本自动执行

### macOS 平台

1. 打开终端
2. 进入项目目录：
   ```bash
   cd /path/to/MMY_SnapMD
   ```
3. 赋予脚本执行权限（首次使用）：
   ```bash
   chmod +x build-mac.sh
   ```
4. 运行脚本：
   ```bash
   ./build-mac.sh
   ```

## 📋 打包流程

脚本会自动执行以下步骤：

1. ✅ 检查 Node.js 是否已安装
2. ✅ 检查 npm 是否已安装
3. ✅ 检查 Rust 是否已安装（Tauri 必需）
4. 📦 安装项目依赖（npm install）
5. 🔨 执行 Tauri 便携版打包（npm run tauri:build:portable）

## 📤 输出文件位置

便携版打包完成后，输出文件位于：

```text
releases\SnapMD-0.1.0-x64.exe
```

此文件可直接双击运行，不需要安装。

安装包打包完成后，输出文件位于：

### Windows
```
MMY_SnapMD\src-tauri\target\release\bundle\
```
包含：
- `.exe` 安装程序（NSIS）
- `.msi` 安装包

当前 Windows 会输出：

```text
MMY_SnapMD\src-tauri\target\release\bundle\msi\SnapMD 闪阅_0.1.0_x64_zh-CN.msi
MMY_SnapMD\src-tauri\target\release\bundle\nsis\SnapMD 闪阅_0.1.0_x64-setup.exe
```

### macOS
```
MMY_SnapMD/src-tauri/target/release/bundle/
```
包含：
- `.app` 应用程序
- `.dmg` 磁盘镜像（如果配置了）

## ❓ 常见问题

### Q: PowerShell 脚本无法运行？
**A:** 修改执行策略：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q: 打包失败，提示 Rust 相关错误？
**A:** 确保已安装 Rust 并重启终端，然后运行：
```bash
rustup update
```

### Q: MSI 打包失败，提示 WiX `LGHT0311` 或 codepage 1252？
**A:** 这是中文产品名与 MSI 默认英文 codepage 不兼容。项目已在 `tauri.conf.json` 中配置 `bundle.windows.wix.language = "zh-CN"`，重新运行：
```powershell
npm run tauri:build
```

### Q: macOS 打包失败，提示代码签名错误？
**A:** 需要在 `src-tauri/tauri.conf.json` 中配置代码签名，或临时禁用：
```json
"bundle": {
  "macOS": {
    "signingIdentity": null
  }
}
```

### Q: 打包很慢？
**A:** 首次打包会下载和编译依赖，需要较长时间（5-15分钟），请耐心等待。

## 📝 注意事项

1. ⚠️ 打包前请确保已安装所有必需的依赖
2. ⚠️ 首次打包时间会较长，后续会利用缓存加快速度
3. ⚠️ 确保磁盘空间充足（至少 2GB 可用空间）
4. ⚠️ 打包过程中请勿关闭终端窗口

## 🎯 高级用法

### 自定义打包配置

编辑 `MMY_SnapMD/src-tauri/tauri.conf.json` 可以自定义：
- 应用名称、版本号
- 安装包格式（NSIS、WiX、DMG等）
- 代码签名配置
- 打包文件路径

详细说明请参考 Tauri 官方文档：
https://tauri.app/v1/api/config/
