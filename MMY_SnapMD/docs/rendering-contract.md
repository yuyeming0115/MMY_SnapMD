# SnapMD 渲染契约

## MVP 支持范围

- 文档类型：
  - `.md` / `.markdown`
  - `.txt`
  - `.json`
  - `.csv`
  - `.yaml` / `.yml`
- CommonMark 基础语法
- GFM 表格、任务列表、删除线、自动链接
- 代码高亮
- 标题锚点与目录
- 图片基础渲染
- 本地 Markdown 文件中的相对图片路径会基于当前文档目录解析
- Tauri 模式下本地图片通过 asset protocol 显示
- 脚注
- TXT 纯文本预览，保留换行、缩进与普通文本符号
- JSON 格式化预览、结构目录与语法错误提示
- CSV 表格预览，首行作为表头，目录显示列信息，最多预览前 500 行数据
- YAML/YML 结构目录、源码预览与基础缩进诊断

## 安全边界

- 原始 HTML 默认关闭
- 渲染输出经过 DOMPurify 清洗
- 不执行脚本、不保留事件属性
- 本地文件访问仅限用户显式打开的文档；Tauri 接入后继续遵守该边界
- 保存仅允许写回显式打开且后缀在白名单内的文本类文档
- 图片资源仅作为图片显示，不作为可执行内容处理

## 导出一致性

- HTML 导出使用同一份渲染 HTML
- PDF 导出优先使用浏览器打印链路
- 打印样式隐藏工具栏与侧边栏，仅输出正文
- 本地相对图片在当前应用内可见；独立 HTML 导出中的本地图片可移植性后续单独处理
- PDF / HTML 导出默认使用源文件基名，例如 `note.md` 对应 `note.pdf` / `note.html`

## 编辑边界

- 编辑区直接修改源文本，预览由源文本派生
- MVP 默认手动保存，不默认自动保存
- 未保存时切换文件需要确认
- 当前文件被外部修改时，若本地已有未保存修改，只提示外部变更，不自动覆盖
- JSON / CSV / YAML 错误诊断不强制阻止保存，但保存前需要确认
- JSON 格式化只在用户显式点击时执行，不自动改写源文本
- 内容字号是显示偏好，不改变文件内容
- Markdown 编辑区的源码高亮仅用于视觉提示，不改变源文本，也不等同于富文本编辑
- CSV / YAML 编辑区的源码高亮仅用于视觉提示，不改变源文本
- 侧栏宽度与分屏比例是界面偏好，不影响文档内容和导出内容
- 工具栏保留阅读、编辑、分屏三个独立模式入口

## 暂不承诺

- 任意 HTML 完整兼容
- 插件注入式扩展语法
- 云分享中的内容审核与访问控制
- Mermaid、数学公式等高级扩展的正式质量保证
