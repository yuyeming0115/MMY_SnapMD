产品定位与核心价值

一款"双击直达、轻量高效、一键分享"的 Markdown 最终效果预览器。解决用户需要快速查看 MD 文件渲染效果，却不得不打开重型编辑器的痛点。

⚡

简单

无繁琐配置，零设置可用

🚀

快速

冷启动 P50 ≤ 700ms

🎯

高效

极低资源占用，高保真排版

🔗

分享

一键私密快照链接

MVP 产品边界（已收敛）

MVP 定义为：一个安装后可关联 .md 文件、稳定渲染常见 Markdown、支持基础主题与导出能力的轻量桌面预览器。

MVP 不包含：多人协作 · 插件系统 · 主题市场 · 小程序 · 企业版 · 账号体系

Phase 1 必做 文件关联 · 渲染 · 代码高亮 · 主题切换 · 实时刷新 · 导出 Phase 2+ 云端分享 · 链接密码 · 移动端H5 · 访问统计

技术路线决策

主路线：Tauri 2 + Vue 3 + TypeScript

  • Markdown 渲染天然受益于 Web 生态
  • 桌面端与分享页可复用渲染规则与样式
  • HTML/CSS 表达力远高于手拼原生控件
  • HTML → Print/PDF 导出链路更易收敛

对照路线：Avalonia 11 + .NET LTS

  • 验证极限冷启动与低内存占用
  • 作为 1 周对照 Spike，非默认落地主线
  • 合理路线：原生控件渲染 MD 或 明确接受 WebView
  • 不推荐"先选 Avalonia 再塞 HTML"的思路

最终拍板建议

产品拍板：先做本地预览器，再验证私密分享，不同时做平台化产品。

技术拍板：以 Tauri 2 作为主路线，以 Avalonia 作为 1 周对照 Spike。

管理拍板：先验收渲染正确性与冷启动，再讨论主题市场和商业化。

5

渲染契约

SnapMD 的本质不是"做一个壳"，而是"稳定呈现 Markdown 最终效果"。渲染契约是产品的口碑底盘。

MVP 承诺

  • CommonMark 基础语法
  • GFM：表格 / 任务列表 / 删除线
  • 代码高亮
  • TOC 自动生成
  • 图片渲染（含相对路径）
  • 引用块 / 脚注

可选支持

  • Mermaid 流程图
  • Frontmatter 元数据
  • 数学公式

暂不承诺

  • 任意内嵌 HTML 完整兼容
  • 插件注入式扩展语法

样本文档集 (render-fixtures)

仓库中建立统一测试样本集，所有技术选型、回归测试、导出验证都基于同一批文件：

basic.md gfm-table.md code-highlight.md long-document.md image-relative-path.md mermaid.md unsafe-html.md print-export.md

SnapMD 差异化定位

"你的 Markdown 秒开预览器，分享只需一个链接"

✅ 双击秒开：比 Typora 快 3–5 倍
✅ 零配置：无需任何设置，安装即用
✅ 一键分享：竞品普遍缺乏的私密快照分享
✅ 导出保真：PDF / HTML 导出与预览一致

分阶段路线图

Phase 0 · 1 周

技术决策 Spike（双栈对照验证）

用 1 周完成 Tauri 与 Avalonia 双栈最小闭环验证。交付双栈样例工程、指标对比表、最终技术决策记录。
决策门槛：渲染正确率 ≥ 90%，冷启动/热切换达标。

Phase 1 · 4–6 周

桌面端 MVP

可日常使用的 Markdown 预览器。文件关联、GFM 渲染、代码高亮、亮/暗主题、文件监听、导出 HTML/PDF。
不进入：云分享、Mermaid 以外复杂扩展、多语言完整覆盖。

Phase 2 · 3–5 周

私密分享 Beta

一键分享链接、H5 只读预览、7 天默认过期、可选密码、手动删除、基础访问统计。
只做私密快照分享，不做公开广场。

Phase 3 · 持续迭代

体验增强

Mermaid 正式支持、多语言、高级主题、最近文件/收藏、移动端阅读优化。

Phase 4 · 后置评估

商业化与平台化

前提：MVP 留存稳定，分享使用率明确，用户出现付费需求。候选：专业版、团队空间、企业部署。

分享信任模型

Phase 2 只做：私密快照分享

  • 分享对象是某次渲染结果快照，不是可协作文档
  • 默认非公开索引，7 天过期
  • 支持可选密码、手动删除、基础访问统计

最小安全规则

HTTPS 强制 链接随机不可枚举 可选密码 7 天过期 手动删除 基础频控
